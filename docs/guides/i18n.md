# Internationalization (i18n)

Making your prestruct site work with multiple languages.

## How Prestruct Handles i18n

Prestruct doesn't have built-in i18n, but you can implement it using standard patterns:

1. **Route-based i18n**: `/en/`, `/es/`, `/fr/` paths
2. **Subdomain i18n**: `en.example.com`, `es.example.com` (requires config)
3. **Query param i18n**: `?lang=es` (not recommended for SEO)

This guide covers route-based i18n, which works best with prestruct.

## Setup

### 1. Define Routes for Each Language

```js
// ssr.config.js
export default {
  siteUrl: 'https://example.com',
  routes: [
    // English (default)
    { path: '/', meta: { title: 'Home | Example', description: 'Welcome to Example' }},
    { path: '/about/', meta: { title: 'About | Example', description: 'Learn about us' }},
    
    // Spanish
    { path: '/es/', meta: { title: 'Inicio | Ejemplo', description: 'Bienvenido a Ejemplo' }},
    { path: '/es/about/', meta: { title: 'Nosotros | Ejemplo', description: 'Conoce más sobre nosotros' }},
    
    // French
    { path: '/fr/', meta: { title: 'Accueil | Exemple', description: 'Bienvenue chez Exemple' }},
    { path: '/fr/about/', meta: { title: 'À propos | Exemple', description: 'En savoir plus sur nous' }},
  ]
}
```

### 2. Create Language Switcher

```jsx
// src/components/LanguageSwitcher.jsx
import { Link, useLocation } from 'react-router-dom'

const languages = [
  { code: 'en', label: 'English', path: '/' },
  { code: 'es', label: 'Español', path: '/es/' },
  { code: 'fr', label: 'Français', path: '/fr/' },
]

export default function LanguageSwitcher() {
  const location = useLocation()
  
  // Get current language from path
  const getCurrentLang = () => {
    const path = location.pathname
    const lang = languages.find(l => path.startsWith(l.path.slice(0, -1)))
    return lang?.code || 'en'
  }
  
  // Get equivalent path in another language
  const getPathForLang = (targetLang) => {
    const currentLang = getCurrentLang()
    let path = location.pathname
    
    // Remove current language prefix
    if (currentLang !== 'en') {
      path = path.replace(`/${currentLang}/`, '/')
    }
    
    // Add new language prefix
    const target = languages.find(l => l.code === targetLang)
    if (targetLang === 'en') {
      return path
    }
    return `/${targetLang}${path === '/' ? '' : path}`
  }
  
  const currentLang = getCurrentLang()
  
  return (
    <div className="language-switcher">
      {languages.map(lang => (
        <Link 
          key={lang.code}
          to={getPathForLang(lang.code)}
          className={lang.code === currentLang ? 'active' : ''}
        >
          {lang.label}
        </Link>
      ))}
    </div>
  )
}
```

### 4. Use in Pages

```jsx
// src/pages/Home.jsx
import usePageMeta from '../hooks/usePageMeta.js'
import { useLocation } from 'react-router-dom'

// Import all locales
import en from './locales/en.js'
import es from './locales/es.js'
import fr from './locales/fr.js'

const locales = { en, es, fr }

function getLang(pathname) {
  if (pathname.startsWith('/es')) return 'es'
  if (pathname.startsWith('/fr')) return 'fr'
  return 'en'
}

export default function Home() {
  const { pathname } = useLocation()
  const lang = getLang(pathname)
  const t = locales[lang]?.home || locales.en.home
  
  usePageMeta({
    path: pathname === '/' ? '/' : pathname,
    title: t.title + ' | Example',
    description: t.subtitle,
  })
  
  return (
    <div>
      <h1>{t.title}</h1>
      <p>{t.subtitle}</p>
      <LanguageSwitcher />
    </div>
  )
}
```

## hreflang for SEO

Add hreflang tags for multilingual SEO:

```jsx
// In your AppLayout or a Head component
useEffect(() => {
  const langs = [
    { lang: 'en', path: '/', url: 'https://example.com/' },
    { lang: 'es', path: '/es/', url: 'https://example.com/es/' },
    { lang: 'fr', path: '/fr/', url: 'https://example.com/fr/' },
  ]
  
  // Remove existing hreflang
  document.querySelectorAll('link[rel="alternate"]').forEach(el => el.remove())
  
  // Add hreflang for current path
  langs.forEach(({ lang, url }) => {
    const link = document.createElement('link')
    link.rel = 'alternate'
    link.hreflang = lang
    link.href = url
    document.head.appendChild(link)
  })
  
  // Add x-default
  const defaultLink = document.createElement('link')
  defaultLink.rel = 'alternate'
  defaultLink.hreflang = 'x-default'
  defaultLink.href = 'https://example.com/'
  document.head.appendChild(defaultLink)
}, [pathname])
```

This generates:

```html
<link rel="alternate" hreflang="en" href="https://example.com/" />
<link rel="alternate" hreflang="es" href="https://example.com/es/" />
<link rel="alternate" hreflang="fr" href="https://example.com/fr/" />
<link rel="alternate" hreflang="x-default" href="https://example.com/" />
```

## Best Practices

### URL Structure

```
example.com/           → English (default)
example.com/es/        → Spanish
example.com/fr/        → French

example.com/about/     → English about
example.com/es/about/  → Spanish about
example.com/fr/about/  → French about
```

### Content Strategy

1. **Keep it simple** - Start with 2-3 languages
2. **Translate meta** - Title, description for each route
3. **Update OG images** - Localized images if needed
4. **Test hreflang** - Use Google Rich Results Test

### Performance

- Use lazy loading for locale files
- Consider JSON instead of JS for translations
- Cache translations in memory

## Error Handling

## Alternatives

### Using react-i18next

```jsx
import i18n from 'i18next'
import { initReactI18next } from 'react-i18next'

i18n
  .use(initReactI18next)
  .init({
    resources: {
      en: { translation: { home: 'Home', about: 'About' }},
      es: { translation: { home: 'Inicio', about: 'Nosotros' }},
    },
    lng: 'en',
    fallbackLng: 'en',
  })

// In your component
const { t } = useTranslation()
return <h1>{t('home')}</h1>
```

**Note:** i18next needs SSR-safe setup for use with prestruct.
